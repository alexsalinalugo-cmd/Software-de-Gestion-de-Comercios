import { IncomingMessage, ServerResponse } from "http"
Importamos los tipos de Node.js nativo. IncomingMessage es el request que llega, ServerResponse es la respuesta que mandamos.

import { CajaService } from "./caja.service"
Traemos el service porque el controller lo necesita para delegar la lógica.

export class CajaController
Creamos la clase y la exportamos para que las routes la puedan usar.

static async abrirCaja(req: IncomingMessage, res: ServerResponse): Promise<void>
La función recibe el request y el response. Devuelve Promise<void>
 porque no devuelve nada al código, sino que responde al navegador con res.end.

 let datos = ""
Creamos una variable vacía donde vamos a ir juntando el body que llega.

req.on("data", (chunk) => { datos += chunk.toString() })
El body no llega de golpe, llega en pedazos llamados chunks. 
Cada vez que llega un pedazo lo convertimos a texto y lo sumamos a datos.

req.on("end", async () => { ... })
Cuando terminaron de llegar todos los pedazos, recién ahí ejecutamos todo lo que está adentro.

const datosParseados = JSON.parse(datos)
datos es un string de texto. Con JSON.parse lo convertimos a un objeto JavaScript que podemos usar.
 Por ejemplo '{"monto_apertura": 5000}' se convierte en { monto_apertura: 5000 }.

 
const caja = await CajaService.abrirCaja(datosParseados)
Le pasamos los datos al service. El await espera a que termine antes de seguir.

res.writeHead(201)
Le decimos al navegador que todo salió bien y que se creó algo nuevo. 201 significa "creado exitosamente".

res.end(JSON.stringify(caja))
Convertimos la caja a texto JSON y la mandamos como respuesta. res.end cierra la respuesta.

catch (error: any)
Si algo falló en el try, caemos acá.
if (error.message === "El monto de apertura debe ser mayor a cero")
Preguntamos si el error es el que nosotros lanzamos en el service. Si es así, es un error del usuario, no del servidor

res.writeHead(400)
400 significa "pedido incorrecto". El usuario mandó datos que no son válidos.


res.end(JSON.stringify({ mensaje: error.message }))
Le devolvemos el mensaje de error para que sepa qué hizo mal.

res.writeHead(500)
Si el error no es el nuestro, es un error del servidor. 500 significa "algo explotó adentro".
